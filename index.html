3. App Permissions and Technical Requirements

The following Android permissions may appear in the app’s manifest due to the way Unreal Engine builds Android applications. These permissions are required by Unreal Engine’s default Android build process and are included automatically when packaging the app. The engine currently does not provide a setting to remove them individually without modifying core build behavior.

  • android.permission.WRITE_EXTERNAL_STORAGE
  • android.permission.READ_EXTERNAL_STORAGE
  • android.permission.READ_MEDIA_AUDIO
  • android.permission.READ_MEDIA_VIDEO
  • android.permission.READ_MEDIA_IMAGES
  • android.permission.ACCESS_MEDIA_LOCATION
  • android.permission.READ_MEDIA_VISUAL_USER_SELECTED

These permissions are present because Android’s build system and Unreal Engine merge multiple required entries by default when external media access is involved.

Important:

  • They are only used for the internal technical operation of the app.
  • They do not result in personal data collection, transmission, or sharing.
  • They are not used to track users or identify individuals.

4. Use of Permissions

If the app requests access at runtime, it is only to satisfy the Android operating system requirements for media access when users explicitly interact with features that would need it (e.g., selecting a file). At no point do we collect or send data beyond the device. The app’s functionality works locally, ensuring privacy‑preserving operation.
